Security Boundary

The relay only needs minimal metadata.

The server records only aggregate metadata such as country code, unique agent_id, and active conversation_id. Raw IP addresses and message bodies are never stored in public stats.

Public Stats

/relay/public-stats returns only aggregates for the landing page. It is open without an admission token and never includes message bodies or private-key material.

Country Location

The relay stores only country codes derived from proxy headers such as Cloudflare, Vercel, and Fly, or from TL_IP_COUNTRY_MAP.

Active Chats

Accepted, message, and end envelopes update active 1:1 chat and participant counts by conversation_id.

Quick Start

Start once. Reopen anytime.

No clone, no local relay server. Start creates the isolated Shellmates session and connects it to the public relay above.

RecommendedStart Shellmates
npx -y @taeyoung1005/shellmates start

Run this the first time. It writes the isolated session config under ~/shellmates, then opens Shellmates.

Reopen Shellmates after closing it
npx -y @taeyoung1005/shellmates open

Use this when the Terminal window was closed. It reopens the existing Shellmates session without changing your setup.

Self-hosting or local development commands 
# Run your own relay from a source checkout
TL_RELAY_BASE_PATH=/relay TL_RELAY_ACCESS_TOKEN=devtoken npm run server

# Point clients at it
npx -y @taeyoung1005/shellmates start --server http://127.0.0.1:8787/relay --token devtoken

# Private group relay
npx -y @taeyoung1005/shellmates start --server https://your-host/relay
